Nuxt.js 学习篇

发表于 2021-06-24

什么是 Nuxt.js

Nuxt.js 是一个基于 Vue.js 的通用应用框架，一个用于 Vue.js 开发 SSR 应用的一站式解决方案。它的优点是将原来几个配置文件要完成的内容，都整合在了一个 nuxt.config.js，封装与扩展性完美的契合。
Nuxtjs 弥补了 Vue 的一个缺点：搜索引擎无法抓取网站内容，不利于 SEO。

什么是 SSR

SSR 又叫做服务器端渲染，前端发出请求后，后端在将 HTML 页面返回给前端之前，先把 HTML 页面中的特定区域，用数据填充好，再将完整的 HTML 返回给前端。在 SPA 场景下，服务端渲染都是针对第一次 get 请求，它会完整的 html 给浏览器，浏览器直接渲染出首屏，用不着浏览器端多一个 AJAX 请求去获取数据再渲染。

SSR 优缺点

优点：
1：更好的 SEO，因为传统的搜索引擎只会从 HTML 中抓取数据，这会导致前端渲染的页面无法被抓取。
2：更快的内容到达时间(time-to-content)，特别是对于缓慢的网络情况或运行缓慢的设备 。页面首屏时间大概有 80%消耗在网络上，剩下的时间在后端读取数据以及浏览器渲染，显然要优化后面的 20%是比较困难的，优化网络时间是效果最明显的手段。传统的 Ajax 请求是先请求 js 再由 js 发起数据请求，两项时间再加上浏览器渲染时间才是首屏时间。而 SSR 能将两个请求合并为一个。
缺点：
1：服务器负载增加；
2：服务器端和浏览器环境差异带来的问题，例如 document 等对象找不到的问题。

异步数据

Nuxt.js 扩展了 Vue.js，增加了一个叫 asyncData 的方法，使得我们可以在设置组件的数据之前能异步获取或处理数据。asyncData 方法会在组件（限于页面组件）每次加载之前被调用。它可以在服务端或路由更新之前被调用。所以需要注意这个函数中不能使用 this。
注意：常规写法如果在 created 钩子中写异步，是在客户端渲染的而不是在服务端。

context.route.params.xxx 获取参数。
callback(new Error(), data)渲染出错的页面。
注意：这个方法在服务器端执行和在客户端执行的区别。

快乐数（简单）

快乐数定义为：
对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
如果 可以变为 1，那么这个数就是快乐数。
如果 n 是快乐数就返回 true ；不是，则返回 false 。

提示：1 <= n <= 231 - 1

我的思路：将数的每位数平方求和，然后重复这个过程直到这个数变为 1，如果为1则返回true，否则返回false。

相同的树（简单）

给你两棵二叉树的根节点 p 和 q ，编写一个函数来检验这两棵树是否相同。
如果两个树在结构上相同，并且节点具有相同的值，则认为它们是相同的。

我的思路：当且仅当两个树完全一致才能返回true，其余皆返回false。

回文数（简单）

给你一个整数 x ，如果 x 是一个回文整数，返回 true ；否则，返回 false 。
回文数是指正序（从左向右）和倒序（从右向左）读都是一样的整数。例如，121 是回文，而 123 不是。

我的思路：将其通过String(x).split(‘’)转为数组，使用数组方法reverse翻转后，使用join方法转为字符串，和原字符串作比较，相同为true，反之为false

发表于 2020-12-21

前言

要了解setTimeout、Promise、Async/Await 的区别，首先我们要知道宏任务与微任务以及他们的执行顺序。
宏任务（macrotask ）和微任务（microtask ）表示异步任务的两种分类。
在挂起任务时，JS 引擎会将所有任务按照类别分到这两个队列中，首先在 macrotask 的队列（这个队列也被叫做 task queue）中取出第一个任务，执行完毕后取出 microtask 队列中的所有任务顺序执行；之后再取 macrotask 任务，周而复始，直至两个队列的任务都取完。

uniapp踩坑之textarea层级问题

发表于 2020-12-05

问题

在开发小程序中，不管是使用原生开发还是使用uniapp、kbone等框架开发，都会遇到一个问题，那就是textarea标签层级过高问题，即使你设置了该标签的层级z-index为最低也无法解决。

解决方法

方法一

当涉及到使用弹出层时，可以当弹出层弹出时，隐藏该textarea标签，即控制该标签的display:none属性

考虑用v-show而不用v-if，是因为v-show只是暂时隐藏掉该文本域，而v-if是销毁掉它（隐藏内容还存在，销毁的话内容不存在）

方法二

为了解决原生组件层级最高的限制。小程序专门提供了 cover-view 和 cover-image 组件，可以覆盖在部分原生组件上面。这两个组件也是原生组件，但是使用限制与其他原生组件有所不同。将定位标签 改为 cover-view标签。但要注意一点就是cover-view不能嵌套cover-image 或者 组件。
